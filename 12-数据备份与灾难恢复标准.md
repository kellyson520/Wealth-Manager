# 数据备份与灾难恢复标准 V1.0

> 适用范围：备份策略、恢复流程、灾难场景、数据完整性

---

## 1. 备份策略

### 1.1 备份层级

```
┌─────────────────────────────────────────┐
│          Level 1: 本地即时备份            │
│  每次数据库写入前自动创建 WAL checkpoint   │
│  保留: 最近 100 个 WAL 帧                 │
│  恢复: 即时，无需用户操作                  │
└─────────────────────────────────────────┘
                    │
┌───────────────────▼─────────────────────┐
│          Level 2: 本地快照备份            │
│  每日凌晨自动创建（充电+WiFi 条件下）       │
│  保留: 最近 7 个快照                      │
│  恢复: 用户手动触发                       │
└─────────────────────────────────────────┘
                    │
┌───────────────────▼─────────────────────┐
│          Level 3: WebDAV 远程备份         │
│  根据配置: 手动 或 定时自动                │
│  保留: 最近 7 个版本                      │
│  加密: AES-256-GCM 端到端加密              │
│  恢复: 换设备 / 灾难恢复                  │
└─────────────────────────────────────────┘
```

### 1.2 备份内容

| 数据类型 | Level 1 (WAL) | Level 2 (本地快照) | Level 3 (WebDAV) |
|----------|:---:|:---:|:---:|
| 账单数据 (bills) | ✅ | ✅ | ✅ |
| 分类规则 (rules) | ✅ | ✅ | ✅ |
| 资产账户 (assets) | ✅ | ✅ | ✅ |
| 记忆数据 (memories) | ✅ | ✅ | ✅ |
| 用户画像 (profile) | ✅ | ✅ | ✅ |
| 审计日志 (audit_log) | ❌ | ✅ | ✅ |
| 审批日志 (approval_log) | ❌ | ✅ | ✅ |
| Skill 数据 | ❌ | ✅ | ✅ |
| 附件图片 | ❌ | ❌ | ✅ |
| 应用设置 | ❌ | ✅ | ✅ |

### 1.3 备份频率

| 触发方式 | 频率 | 条件 |
|----------|------|------|
| **WAL 自动** | 每次写入 | 无需条件 |
| **本地快照** | 每日 1 次 | 充电 + WiFi + 屏幕关闭 |
| **WebDAV 自动** | 每 N 小时 (默认 24) | 充电 + WiFi + WebDAV 已配置 |
| **WebDAV 手动** | 用户随时触发 | WebDAV 已配置 |

## 2. WebDAV 备份详细规范

### 2.1 加密流程

```
原始数据
  │
  ▼
Step 1: 打包
  将所有数据 + 附件图片压缩为 .tar.gz
  │
  ▼
Step 2: 加密
  生成随机 AES-256 数据密钥 (DEK)
  使用 DEK 加密压缩包 (AES-256-GCM)
  │
  ▼
Step 3: 密钥加密
  使用用户备份密码 → PBKDF2 (迭代 600,000 次) → 密钥加密密钥 (KEK)
  使用 KEK 加密 DEK
  │
  ▼
Step 4: 签名
  SHA-256(密文 + 加密的 DEK)
  │
  ▼
Step 5: 组装
  ┌──────────────────────────────┐
  │ 加密备份包结构:               │
  │  [Header]                    │
  │    - version: 1              │
  │    - algorithm: AES-256-GCM  │
  │    - kdf: PBKDF2-SHA256      │
  │    - salt: <随机盐>           │
  │  [Encrypted DEK]             │
  │  [IV + Ciphertext + Tag]     │
  │  [Signature: SHA-256]        │
  └──────────────────────────────┘
  │
  ▼
上传到 WebDAV: /bill_backup/backup_2025-01-15T08_00_00Z.wmenc
```

### 2.2 文件命名规范

```
backup_{YYYY}-{MM}-{DD}T{HH}_{MM}_{SS}Z.wmenc

示例:
  backup_2025-01-15T08_00_00Z.wmenc    (手动)
  backup_2025-01-15T03_00_00Z_auto.wmenc  (自动)
```

### 2.3 增量同步

```
首次备份: 全量上传
  │
  ▼
后续备份:
  1. 计算本地数据变更 (对比上次快照的哈希)
  2. 仅打包变更的数据表
  3. 上传增量包 (后缀: _inc.wmenc)
  4. 每 7 次增量 → 创建新的全量快照
```

### 2.4 版本保留与清理

| 规则 | 说明 |
|------|------|
| 保留最近 7 个版本 | 全量 + 增量均算在内 |
| 自动清理 | 上传新版本时，自动删除第 8 个及更旧的版本 |
| 手动锁定 | 用户可锁定某个版本，永不过期 |
| 存储估算 | 7 个版本 ≈ 数据库大小 × 3 (含压缩和历史变化) |

## 3. 恢复流程

### 3.1 同设备恢复 (本地快照)

```
用户操作: 设置 → 数据 → 恢复 → 选择快照
  │
  ▼
Step 1: 验证快照完整性 (SHA-256)
  │ 失败 → 提示 "快照已损坏"
  │
  ▼
Step 2: 创建当前数据备份 (安全措施)
  │
  ▼
Step 3: 清空当前数据库
  │
  ▼
Step 4: 导入快照数据
  │
  ▼
Step 5: 验证哈希链完整性
  │
  ▼
Step 6: App 重启 → 加载恢复后的数据
```

### 3.2 跨设备恢复 (WebDAV)

```
新设备首次启动
  │
  ▼
Step 1: 用户选择 "从备份恢复"
  │
  ▼
Step 2: 输入 WebDAV 地址 + 用户名 + 密码
  │
  ▼
Step 3: test_webdav_connection → 列出可用备份
  │
  ▼
Step 4: 用户选择备份版本
  │
  ▼
Step 5: 下载加密包
  │
  ▼
Step 6: 输入备份密码 → 解密
  │
  ▼
Step 7: 验证签名
  │ 失败 → "备份文件可能被篡改或损坏"
  │
  ▼
Step 8: 导入数据
  │
  ▼
Step 9: 验证哈希链完整性
  │ 失败 → "数据完整性异常，可能部分账单丢失"
  │
  ▼
Step 10: 恢复完成 → 正常使用
```

### 3.3 恢复时间目标 (RTO)

| 场景 | 数据量 | 预计时间 |
|------|--------|----------|
| WAL 恢复 | — | < 1s |
| 本地快照恢复 | 100MB | < 10s |
| WebDAV 恢复 (WiFi) | 50MB | < 30s |
| WebDAV 恢复 (WiFi) | 500MB | < 3min |
| WebDAV 恢复 (蜂窝) | 50MB | < 2min (取决于信号) |

## 4. 灾难场景

### 4.1 场景与应对

| 灾难场景 | 检测方式 | 恢复方案 | RTO |
|----------|----------|----------|-----|
| **数据库文件损坏** | 启动时 PRAGMA integrity_check | 从最近本地快照恢复 | < 10s |
| **SQLite WAL 损坏** | 打开数据库时报错 | 自动丢弃 WAL，从主 DB 恢复 | < 1s |
| **单条账单哈希断裂** | verify_hash_chain 检测 | repair_hash_chain 修复 | < 1s |
| **批量哈希断裂** | verify_hash_chain 检测 | 从最近快照恢复断裂点之后的数据 | < 30s |
| **手机丢失/损坏** | — | 新设备从 WebDAV 恢复 | < 3min |
| **WebDAV 服务器不可用** | 连接超时 | 使用本地快照 + 等待服务器恢复 | 取决于服务器 |
| **备份密码遗忘** | — | **无法恢复**（设计如此，端到端加密） | ∞ |
| **勒索软件加密本地文件** | 应用无法启动 | 重装 App → 从 WebDAV 恢复 | < 5min |
| **WebDAV 备份被删除** | 远程列表为空 | 使用本地快照 + 重新上传 | < 1min |

### 4.2 备份密码丢失处理

```
用户遗忘备份密码
  │
  ▼
提示: "备份密码无法找回。这是为了确保只有你能解密数据。"
  │
  ▼
选项:
  A) 尝试其他密码 (无限制)
  B) 重置备份: 生成新密码 → 创建新备份 → 旧备份永久不可访问
  C) 查看密码提示 (用户在创建备份时可选设置)
```

### 4.3 数据完整性验证

```
定期自动验证 (每周日凌晨):
  │
  ▼
1. verify_hash_chain()
  ├─ 完整 → 静默通过
  └─ 断裂 → 通知用户 + 触发修复向导
  │
  ▼
2. 数据库完整性检查 (PRAGMA integrity_check)
  ├─ 通过 → 静默通过
  └─ 失败 → 尝试从快照恢复
  │
  ▼
3. 备份有效性检查
  └─ 最近一次 WebDAV 备份是否可成功恢复？
      ├─ 是 → 静默通过
      └─ 否 → 通知用户 "备份异常，请检查配置"
```

## 5. 备份安全

### 5.1 加密参数

| 参数 | 值 |
|------|-----|
| 加密算法 | AES-256-GCM |
| 密钥派生 | PBKDF2-HMAC-SHA256 |
| 迭代次数 | 600,000 (OWASP 2025 推荐) |
| 盐长度 | 256 bits (随机生成) |
| IV 长度 | 96 bits (随机生成) |
| GCM Tag 长度 | 128 bits |

### 5.2 密钥管理

```
备份密码
  │
  ▼
PBKDF2 (600,000 迭代, 随机盐)
  │
  ▼
密钥加密密钥 (KEK) ─── 仅存于内存，使用后清零
  │
  ▼
加密/解密 数据加密密钥 (DEK)
  │
  ▼
DEK ─── 仅存于内存，使用后清零
  │
  ▼
加密/解密 备份数据
```

### 5.3 安全注意事项

| 注意点 | 说明 |
|--------|------|
| **密码绝不存储** | 备份密码不存储在设备上，每次恢复需重新输入 |
| **KEK 内存清零** | 加密/解密完成后立即 `memset(0)` 覆盖密钥 |
| **DEK 一次性** | 每次备份使用新的随机 DEK |
| **传输安全** | WebDAV 必须通过 HTTPS 传输 |
| **证书验证** | 严格验证 TLS 证书，拒绝自签名 |

## 6. 备份监控

| 指标 | 监控方式 | 告警条件 |
|------|----------|----------|
| 上次备份距今 | 时间比较 | > 自动间隔 × 1.5 |
| 连续备份失败 | 计数器 | ≥ 3 次 |
| 备份文件大小异常 | 与上次对比 | 变化 > 50% 且无正当理由 |
| 哈希链完整性 | 定期验证 | 检测到断裂 |
| WebDAV 可用空间 | 配额查询 | 不足 2 个备份包大小 |
