## 1. Agent 间消息协议

### 1.1 消息总线架构

```
┌─────────────────────────────────────────────┐
│              Message Bus (消息总线)           │
│                                             │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐     │
│  │ Master  │  │ Ledger  │  │ Analyst │ ... │
│  └────┬────┘  └────┬────┘  └────┬────┘     │
│       │            │            │           │
│       ▼            ▼            ▼           │
│  ┌─────────────────────────────────────┐    │
│  │        Message Router (路由器)       │    │
│  │  - 点对点路由                        │    │
│  │  - 广播 (仅安全事件)                 │    │
│  │  - 请求/响应关联                     │    │
│  └─────────────────────────────────────┘    │
└─────────────────────────────────────────────┘
```

### 1.2 消息格式

```typescript
interface AgentMessage {
  // 消息元数据
  messageId: string;           // UUID v4
  timestamp: string;           // ISO 8601
  ttl: number;                 // 生存时间 (ms)，默认 30000

  // 路由信息
  source: AgentId;             // 发送方
  target: AgentId | 'broadcast'; // 接收方
  replyTo?: string;            // 关联的请求 messageId
  
  // 消息体
  type: MessageType;
  payload: MessagePayload;
  
  // 安全
  priority: 'normal' | 'high' | 'critical';
  traceId: string;             // 全链路追踪 ID
}

type AgentId = 'master' | 'ledger' | 'analyst' | 'coach' | 'guardian';

type MessageType = 
  | 'intent.classify'       // 意图分类请求
  | 'intent.result'         // 意图分类结果
  | 'task.execute'          // 任务执行请求
  | 'task.result'           // 任务执行结果
  | 'task.progress'         // 任务进度更新
  | 'safety.check'          // 安全检查请求
  | 'safety.veto'           // 安全否决
  | 'safety.confirm_card'   // 插入确认卡片
  | 'context.query'         // 上下文查询
  | 'context.update'        // 上下文更新
  | 'system.event'          // 系统事件广播
  | 'system.error';         // 系统错误
```

### 1.3 典型消息流

**记账流程：**

```
User → Master: "午饭花了35块"
  │
  ▼
Master → MessageBus: { type: "intent.classify", payload: { text: "午饭花了35块" } }
  │
  ▼ (本地 NLU 分类 + 必要时云端)
Master: 意图 = record_bill, 参数 = { merchant: "午饭", amount: 35 }
  │
  ▼
Master → Ledger: { type: "task.execute", payload: { tool: "add_bill", params: {...} } }
  │
  ▼
Ledger → Guardian: { type: "safety.check", payload: { tool: "add_bill", params: {...} } }
  │
  ├─ 通过 → Ledger 执行 → Master → User: "已记录：午饭 ¥35.00"
  └─ 拦截 → Guardian → Master → User: 确认卡片
```

### 1.4 消息传递保证

| 保证项 | 实现 |
|--------|------|
| 消息不丢失 | 本地内存队列 + 持久化到 SQLite（关键消息） |
| 消息不重复 | messageId 去重，幂等处理 |
| 消息有序 | 同一 traceId 的消息按时间戳排序 |
| 超时处理 | 默认 30s TTL，超时自动返回 timeout 错误 |

## 2. 工具注册与调用协议

### 2.1 工具注册接口

```typescript
interface ToolRegistry {
  // 注册工具
  register(tool: ToolDefinition): void;
  
  // 查询工具（供 Agent 发现可用工具）
  listByNamespace(namespace: string): ToolDefinition[];
  listByAgent(agentId: AgentId): ToolDefinition[];
  listByPermission(maxLevel: 0 | 1 | 2): ToolDefinition[];
  
  // 调用工具
  invoke(request: ToolInvocation): Promise<ToolResult>;
  
  // 工具元数据
  getTool(name: string): ToolDefinition | null;
  hasTool(name: string): boolean;
}
```

### 2.2 工具调用流程

```
Agent → ToolRegistry.invoke({ toolName, params, agentId })
  │
  ▼
1. 权限校验
   └─ 工具权限级别 vs Agent 权限上限
   └─ 不通过 → 返回 { error: "permission_denied" }
  │
  ▼
2. 参数校验 (JSON Schema)
   └─ 不通过 → 返回 { error: "invalid_params", details: [...] }
  │
  ▼
3. Guardian 安全检查
   └─ 0级 → 跳过
   └─ 1级/2级 → 风险评估 → 可能插入确认卡片
  │
  ▼
4. 执行工具
   └─ 超时 5s → 返回 { error: "timeout" }
   └─ 异常 → 返回 { error: "execution_error", ... }
  │
  ▼
5. 审计记录
   └─ 写入 audit_log
  │
  ▼
6. 返回结果
```

### 2.3 工具调用请求/响应

```typescript
interface ToolInvocation {
  invocationId: string;        // UUID v4
  toolName: string;            // 如 "bills.add_bill"
  params: Record<string, unknown>;
  agentId: AgentId;
  traceId: string;
  timeout: number;             // 默认 5000ms
}

interface ToolResult {
  invocationId: string;
  status: 'success' | 'error' | 'blocked' | 'timeout';
  data?: unknown;
  error?: {
    code: string;
    message: string;
    retryable: boolean;
    details?: unknown;
  };
  durationMs: number;
}
```

## 3. 云端 AI API 集成

### 3.1 支持的模型

| 模型 | 用途 | 优先级 |
|------|------|--------|
| GPT-4o | 复杂推理、深度分析 | 主模型 |
| Claude 3.5 Sonnet | 替代模型 | 备用 |
| GPT-4o-mini | 简单分类、规则生成 | 降级 |
| 本地端侧模型 (可选) | 离线场景 | 兜底 |

### 3.2 请求格式

```typescript
interface CloudAIRequest {
  model: string;
  messages: {
    role: 'system' | 'user' | 'assistant';
    content: string;
  }[];
  tools?: ToolDefinition[];    // Function Calling 工具定义
  temperature?: number;        // 默认 0.3 (偏精确)
  max_tokens?: number;         // 默认 2000
  response_format?: { type: 'json_object' };
}

interface CloudAIResponse {
  id: string;
  model: string;
  choices: {
    message: {
      role: 'assistant';
      content: string | null;
      tool_calls?: ToolCall[];
    };
    finish_reason: 'stop' | 'tool_calls' | 'length';
  }[];
  usage: {
    prompt_tokens: number;
    completion_tokens: number;
    total_tokens: number;
  };
}
```

### 3.3 发送前处理

每次云端请求前，必须经过脱敏管道：

```
原始请求
  │
  ▼
sanitize_for_cloud()
  ├─ 金额 → 区间桶
  ├─ 商户名 → 分类标签
  ├─ 金额序列 → 聚合统计
  ├─ 银行名 → 匿名标签
  └─ 精确日期 → 模糊日期
  │
  ▼
Token 预算检查
  ├─ < 80% → 正常发送
  ├─ ≥ 80% → 发送 + 用户警告
  └─ ≥ 100% → 阻断，降级本地
  │
  ▼
发送 (TLS 1.3 + 证书绑定)
```

### 3.4 降级策略

| 场景 | 降级方案 |
|------|----------|
| 网络不可用 | 本地 NLU + 规则引擎处理 |
| API 超时 (10s) | 本地兜底处理 |
| API 返回错误 | 重试 1 次 → 失败则本地处理 |
| Token 耗尽 | 阻断云端，纯本地模式 |
| 用户开启离线模式 | 永久本地模式 |

## 4. WebDAV 集成

### 4.1 协议适配层

```typescript
interface WebDAVAdapter {
  // 连接测试
  testConnection(config: WebDAVConfig): Promise<ConnectionTestResult>;
  
  // 文件操作
  upload(path: string, data: ArrayBuffer, options: UploadOptions): Promise<void>;
  download(path: string): Promise<ArrayBuffer>;
  list(path: string): Promise<WebDAVFile[]>;
  delete(path: string): Promise<void>;
  
  // 配额查询
  getQuota(): Promise<{ used: number; available: number }>;
}

interface WebDAVConfig {
  url: string;                  // https://dav.example.com
  username: string;
  passwordEncrypted: string;    // AES-256 加密存储
  remotePath: string;           // 默认 /bill_backup
}
```

### 4.2 服务商兼容性矩阵

| 服务商 | 速率限制 | 最大文件 | 特殊处理 |
|--------|----------|----------|----------|
| 坚果云 | 600次/30min | 无限制 | 需 User-Agent 头 |
| Nextcloud | 无硬限制 | 取决于配置 | 支持分块上传 |
| ownCloud | 无硬限制 | 取决于配置 | 标准 WebDAV |
| 自建 Apache | 取决于配置 | 取决于配置 | 标准 WebDAV |
| Synology NAS | 无硬限制 | 无限制 | 需处理特殊字符路径 |

### 4.3 智能适配策略

| 适配项 | 策略 |
|--------|------|
| 速率控制 | 初次连接自动探测服务商 → 应用对应速率策略 |
| 错误重试 | 429/503 → 指数退避 (1s → 2s → 4s → 8s)，最多 4 次 |
| 分块上传 | 文件 > 100MB → 拆分为 10MB 块，并发上传 3 块 |
| 仅 WiFi | 检测网络类型，移动网络下暂停自动备份 |
| 冲突处理 | 远程有更新版本 → 下载合并 → 重新上传 |

## 5. Siri / Google Assistant 集成

### 5.1 Siri Intent 定义

| Intent | 参数 | 示例 |
|--------|------|------|
| `RecordExpense` | amount, merchant, category? | "记一笔午饭 35 块" |
| `QueryBills` | category?, dateRange? | "查这个月餐饮花了多少" |
| `GetMonthlyReport` | month? | "这个月花了多少" |
| `GetBudgetStatus` | category? | "餐饮预算还剩多少" |

### 5.2 Siri Intent 处理

```
Siri → App Intent Handler
  │
  ▼
解析参数 → 构造内部请求 → Master Agent
  │
  ▼
返回结果 → Siri 语音播报 (精简版)
```

```

  ▼
接收交易文本/截图
  │
  ▼
OCR 解析 (截图) / 文本解析 → 预填记账表单
  │
  ▼
用户确认 → 记账完成
```

## 7. 接口版本管理

| 接口 | 版本策略 | 兼容性 |
|------|----------|--------|
| Agent 消息协议 | 语义版本 (MAJOR.MINOR) | MINOR 升级向后兼容 |
| 工具注册接口 | 工具级版本（每个工具独立版本） | 旧版本保留 2 个大版本 |
| 云端 API | 跟随模型版本 | 由 prompt 版本控制 |
| WebDAV | HTTP 标准，无版本 | 协议级兼容 |
| Siri Intent | 跟随 OS 版本 | Apple 定义 |

---

## 8. 接口安全

| 接口层面 | 安全措施 |
|----------|----------|
| Agent 间通信 | 仅本地内存，不跨进程 |
| 工具调用 | 权限级别 + Guardian 检查 + 审计 |
| 云端 API | TLS 1.3 + 证书绑定 + 数据脱敏 |
| WebDAV | HTTPS + AES-256 加密 + 速率限制 |
| Siri | 系统级沙盒，仅传递 Intent 参数 |
| 分享扩展 | 应用组共享容器，不直接访问数据库 |
