### 1.2 文案管理

```typescript
// 使用 i18next 或自建方案
// 文案文件: locales/{lang}/common.json

// 结构示例
{
  "bill": {
    "add_success": "已记录：{{merchant}} ¥{{amount}}",
    "add_success_plural": "已记录 {{count}} 笔账单",

    "emergency_triggered": "已触发紧急熔断，当前操作已回滚。"
  },
  "chart": {
    "monthly_expense": "{{month}}月支出",
    "category_breakdown": "分类构成"
  }
}
```

### 1.3 翻译原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **语境完整** | Key 名要体现语境，不依赖代码位置 | ✅ `bill.delete_confirm` ❌ `confirm_delete` |
| **避免拼接** | 不拆句，用插值 | ✅ `"{{merchant}}: ¥{{amount}}"` ❌ `merchant + ": ¥" + amount` |
| **单复数** | 使用 i18n 复数规则 | `_plural` 后缀 |
| **语气一致** | 中文偏亲切，英文偏简洁 | 中文"记一笔？" 英文 "Add?" |
| **金融术语准确** | 核对会计术语 | refund→退款(非"退还")，balance→余额(非"平衡") |

### 1.4 AI 回复多语言

AI 回复默认跟随 App 界面语言，用户可手动切换 AI 回复语言：

| 场景 | 处理 |
|------|------|
| 用户输入中文 | AI 用中文回复 |
| 用户输入英文 | AI 用英文回复 |
| 用户混合中英 | AI 跟随最后使用的语言 |
| 用户设定 AI 语言 | 始终使用设定语言 |

目标：Phase 5 支持 ≥ 50 种货币（ISO 4217 标准）。

### 2.2 金额存储规范

```typescript
// 内部存储：始终以最小单位 (整数) 存储
interface Money {
  amount: number;         // 整数，如 123456 表示 ¥1,234.56
  currency: string;       // ISO 4217 货币代码
}

// 显示时根据货币的 decimalPlaces 格式化
const CURRENCY_CONFIG: Record<string, CurrencyConfig> = {
  CNY: { symbol: '¥', decimalPlaces: 2, thousandsSep: ',', decimalSep: '.' },
  USD: { symbol: '$', decimalPlaces: 2, thousandsSep: ',', decimalSep: '.' },
  JPY: { symbol: '¥', decimalPlaces: 0, thousandsSep: ',', decimalSep: '.' },
  KRW: { symbol: '₩', decimalPlaces: 0, thousandsSep: ',', decimalSep: '.' },
};
```

### 2.3 汇率处理

| 特性 | 实现 |
|------|------|
| 汇率数据源 | 本地缓存 + 每日自动更新 (开放汇率 API) |
| 离线可用 | 使用最近一次缓存的汇率 |
| 换算精度 | 中间计算保留 6 位小数，最终显示按货币 decimalPlaces |
| 默认显示 | 以主货币显示，其他货币标注换算值 |

## 3. 地区适配

### 3.1 日期时间

| 地区 | 日期格式 | 示例 | 周起始日 |
|------|----------|------|----------|
| 中国大陆 | yyyy年M月d日 | 2025年1月15日 | 周一 |
| 美国 | M/d/yyyy | 1/15/2025 | 周日 |
| 英国 | d/M/yyyy | 15/1/2025 | 周一 |
| 日本 | yyyy年M月d日 | 2025年1月15日 | 周一 |

### 3.2 数字格式

| 地区 | 千分位 | 小数点 | 示例 |
|------|--------|--------|------|
| 中国 | , | . | ¥1,234.56 |
| 欧洲 (部分) | . | , | 1.234,56 € |
| 印度 | , (分组不同) | . | ₹1,23,456.78 |

### 3.3 会计习惯差异

| 差异点 | 中国大陆 | 西方 |
|--------|----------|------|
| 收入表述 | "+收入" | "Income" |
| 颜色语义 | 红色=支出(赤字) | 红色=亏损，绿色=盈利 |
| 结余计算 | 收入 - 支出 | Income - Expense |
| 分类习惯 | 餐饮、交通、购物 | Food, Transportation, Shopping |

## 4. 本地法规合规

### 4.1 数据隐私法规

| 法规 | 适用范围 | 核心要求 |
|------|----------|----------|
| 《个人信息保护法》(PIPL) | 中国大陆用户 | 数据本地化、单独同意、可携带 |
| GDPR | EU/EEA 用户 | 数据最小化、删除权、DPO |
| CCPA/CPRA | 加州用户 | 知情权、删除权、不出售数据 |

### 4.2 合规实现

| 要求 | 实现 |
|------|------|
| 数据处理告知 | 首次启动隐私政策弹窗，需主动同意 |
| 数据导出 | 支持 JSON/CSV/加密包三种格式导出 |
| 数据删除 | `clear_all_data` 物理删除 + 内存覆盖 |
| 云端数据处理 | 默认不上传，需单独授权 |
| 数据传输记录 | 每次云端调用记录到 audit_log |
| 隐私设置面板 | 统一管理所有隐私选项 |

### 4.3 App Store 隐私标签
