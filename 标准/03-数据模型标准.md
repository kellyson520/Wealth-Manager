# 数据模型标准 V1.0

> 来源：AI 原生对话式记账系统 最终汇总方案 V4.0
> 适用范围：数据库设计、数据迁移、存储规范

---

## 1. 数据库选型

| 属性 | 要求 |
|------|------|
| 引擎 | SQLite (加密) |
| 加密方式 | SQLCipher 或等效方案 |
| 向量存储 | 本地向量库（与 SQLite 分离） |
| 迁移策略 | 版本化迁移脚本，向前兼容 |

## 2. 核心表结构

### 2.1 账单表 (bills)

```sql
CREATE TABLE bills (
  id TEXT PRIMARY KEY,
  amount REAL NOT NULL,
  type TEXT NOT NULL CHECK(type IN ('income','expense','refund')),
  category_id TEXT,
  tags TEXT,
  merchant TEXT,
  raw_description TEXT,
  date TEXT NOT NULL,
  note TEXT,
  emotion_score INT,
  images TEXT,
  source TEXT DEFAULT 'manual',
  refund_original_id TEXT,
  hash_chain TEXT NOT NULL,
  previous_hash TEXT,
  created_at TEXT NOT NULL
);
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | TEXT | PK | UUID v4 |
| amount | REAL | NOT NULL | 金额，正数 |
| type | TEXT | NOT NULL, CHECK | income / expense / refund |
| category_id | TEXT | FK → categories | 分类 ID |
| tags | TEXT | | JSON 数组字符串 |
| merchant | TEXT | | 商家名称 |
| raw_description | TEXT | | 原始描述（用户输入原文） |
| date | TEXT | NOT NULL | ISO 8601 日期 |
| note | TEXT | | 用户备注 |
| emotion_score | INT | | 情绪分数 1-5 |
| images | TEXT | | 图片路径 JSON 数组 |
| source | TEXT | DEFAULT 'manual' | manual / import / auto / ocr |
| refund_original_id | TEXT | FK → bills.id | 退款关联的原账单 |
| hash_chain | TEXT | NOT NULL | 链式哈希值 |
| previous_hash | TEXT | | 前一条账单的 hash_chain |
| created_at | TEXT | NOT NULL | 创建时间戳 |

### 2.2 分类规则表 (classification_rules)

```sql
CREATE TABLE classification_rules (
  id TEXT PRIMARY KEY,
  pattern TEXT NOT NULL,
  category_id TEXT,
  merchant TEXT,
  tags TEXT,
  confidence REAL DEFAULT 1.0
);
```

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT | PK |
| pattern | TEXT | 匹配模式（正则或关键词） |
| category_id | TEXT | 目标分类 |
| merchant | TEXT | 商家名 |
| tags | TEXT | 自动附加标签 |
| confidence | REAL | 置信度 0.0-1.0 |

### 2.3 资产账户表 (asset_accounts)

```sql
CREATE TABLE asset_accounts (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  type TEXT NOT NULL CHECK(type IN ('cash','bank','credit','investment','other')),
  balance REAL NOT NULL,
  currency TEXT DEFAULT 'CNY',
  updated_at TEXT NOT NULL
);
```

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT | PK |
| name | TEXT | 账户名称 |
| type | TEXT | cash / bank / credit / investment / other |
| balance | REAL | 当前余额 |
| currency | TEXT | 货币代码，默认 CNY |
| updated_at | TEXT | 最后更新时间 |

### 2.4 记忆表 (memories)

```sql
CREATE TABLE memories (
  id TEXT PRIMARY KEY,
  type TEXT NOT NULL CHECK(type IN ('long_term','episodic')),
  content TEXT NOT NULL,
  embedding BLOB,
  updated_at TEXT
);
```

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT | PK |
| type | TEXT | long_term（长期偏好）/ episodic（情景记忆） |
| content | TEXT | 记忆内容（自然语言） |
| embedding | BLOB | 向量嵌入（用于相似度检索） |
| updated_at | TEXT | 最后更新时间 |

### 2.5 用户画像表 (user_profile)

```sql
CREATE TABLE user_profile (
  id TEXT PRIMARY KEY DEFAULT 'singleton',
  persona_params TEXT,
  budget_limits TEXT,
  preferences TEXT

```

| 字段 | 类型 | 说明 |
|------|------|------|

| persona_params | TEXT | JSON：严谨度/幽默度/主动度 |
| budget_limits | TEXT | JSON：各分类预算上限 |
| preferences | TEXT | JSON：用户偏好设置 |

### 2.6 WebDAV 备份配置表 (backup_config)

```sql
CREATE TABLE backup_config (
```

| 字段 | 类型 | 说明 |
|------|------|------|

| remote_path | TEXT | 远程备份路径 |

| backup_interval_hours | INTEGER | 自动备份间隔，默认 24h |
| last_backup_time | TEXT | 上次备份时间 |
| last_backup_status | TEXT | 上次备份状态 |

## 3. 审计日志表 (audit_log)

```sql
CREATE TABLE audit_log (
  id TEXT PRIMARY KEY,
  timestamp TEXT NOT NULL,
  agent_id TEXT NOT NULL,
  tool_name TEXT NOT NULL,
  params_hash TEXT NOT NULL,
  result_status TEXT NOT NULL CHECK(result_status IN ('success','error','blocked')),
  permission_level INT NOT NULL,
  user_confirmed INTEGER DEFAULT 0,
  duration_ms INT
);
```

- 保留周期：30 天
- 支持查询和导出
- 不可手动修改或删除

## 4. 数据完整性约束

### 4.1 哈希链约束

- 每条账单必须包含 `hash_chain` 和 `previous_hash`
- `hash_chain = SHA256(previous_hash + bill_data + secret_salt)`
- 插入新账单时必须验证上一条的哈希链
- 检测到断裂时触发 `repair_hash_chain`

### 4.2 外键约束

| 表 | 字段 | 引用 |
|----|------|------|
| bills.refund_original_id | → | bills.id |
| bills.category_id | → | categories.id |
| classification_rules.category_id | → | categories.id |

### 4.3 数据迁移规范

- 每次 schema 变更必须有独立的迁移脚本
- 迁移脚本命名：`{YYYYMMDDHHMMSS}_{description}.sql`
- 迁移必须在事务中执行，失败则回滚
- 迁移前自动创建备份快照

## 5. 加密存储规范

| 数据类型 | 加密方式 | 说明 |
|----------|----------|------|
| SQLite 数据库 | SQLCipher 全库加密 | 用户设置数据库密码 |
| WebDAV 备份 | AES-256-GCM | 端到端加密，密码由用户设定 |
| WebDAV 密码 | AES-256 加密存储 | 使用设备密钥派生加密密钥 |
| 云端传输 | TLS 1.3 | 传输层加密 |
| 向量嵌入 | 明文存储 | 无需加密（不可逆） |

---

## 6. 数据生命周期

| 阶段 | 策略 |
|------|------|
| 创建 | 即时刻录哈希链 |
| 修改 | 追加新版本，旧版本保留 30 天 |
| 删除 | 软删除（标记 deleted_at），30 天后物理删除 |
| 归档 | 超过 1 年的数据可选归档到备份 |
| 导出 | 支持 CSV / JSON / 加密包三种格式 |
| 清除 | `clear_all_data` 需二次确认，物理删除不可恢复 |

## 7. 索引与查询优化

### 7.1 推荐索引

```sql
-- 账单表：按时间和分类查询最频繁
CREATE INDEX idx_bills_date ON bills(date);
CREATE INDEX idx_bills_category ON bills(category_id);
CREATE INDEX idx_bills_merchant ON bills(merchant);
CREATE INDEX idx_bills_type_date ON bills(type, date);
CREATE INDEX idx_bills_source ON bills(source);
CREATE INDEX idx_bills_hash_chain ON bills(hash_chain);

-- 分类规则表：按匹配模式查询
CREATE INDEX idx_rules_pattern ON classification_rules(pattern);
CREATE INDEX idx_rules_category ON classification_rules(category_id);
CREATE INDEX idx_rules_confidence ON classification_rules(confidence);

-- 审计日志表：按时间和工具名查询
CREATE INDEX idx_audit_timestamp ON audit_log(timestamp);
CREATE INDEX idx_audit_tool ON audit_log(tool_name);
CREATE INDEX idx_audit_agent ON audit_log(agent_id);

-- 审批日志表：按时间和工具名查询
CREATE INDEX idx_approval_created ON approval_log(created_at);
CREATE INDEX idx_approval_tool ON approval_log(tool_name);
```

### 7.2 查询性能基线

| 查询类型 | SQL 示例 | 目标 (10万条数据) |
|----------|----------|-------------------|
| 按日期范围 | `WHERE date BETWEEN ? AND ?` | < 50ms |
| 按分类聚合 | `GROUP BY category_id` | < 100ms |
| 商户模糊搜索 | `WHERE merchant LIKE '%星巴克%'` | < 200ms |
| 全文搜索 | 使用 FTS5 虚拟表 | < 100ms |
| 哈希链验证 | 遍历全表计算 | < 500ms |
| 多条件组合 | 日期 + 分类 + 金额范围 | < 150ms |

### 7.3 全文搜索 (FTS5)

```sql
-- 创建 FTS5 虚拟表用于账单搜索
CREATE VIRTUAL TABLE IF NOT EXISTS bills_fts USING fts5(
  merchant,
  raw_description,
  note,
  tags,
  content=bills,
  content_rowid=rowid
);

-- 同步触发器
CREATE TRIGGER bills_ai AFTER INSERT ON bills BEGIN
  INSERT INTO bills_fts(rowid, merchant, raw_description, note, tags)
  VALUES (new.rowid, new.merchant, new.raw_description, new.note, new.tags);
END;

-- 搜索示例
SELECT * FROM bills_fts WHERE bills_fts MATCH '星巴克 OR 咖啡';
```

### 7.4 WAL 模式

```sql
-- 启用 WAL 模式以支持并发读写
PRAGMA journal_mode = WAL;
PRAGMA wal_autocheckpoint = 1000;  -- 每 1000 页自动 checkpoint
PRAGMA synchronous = NORMAL;       -- 平衡性能与安全
PRAGMA cache_size = -8000;         -- 8MB 缓存
PRAGMA foreign_keys = ON;
PRAGMA mmap_size = 268435456;      -- 256MB 内存映射
```
