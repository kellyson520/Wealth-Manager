# 部署与运维标准 V1.0

> 适用范围：CI/CD、构建发布、环境配置、监控告警、数据库迁移

---

## 1. CI/CD 流水线

### 1.1 流水线总览

```
Git Push (main / feature/*)
  │
  ▼
┌──────────────────────────────────────────────┐
│ Stage 1: 静态检查 (2 min)                      │
│  - TypeScript strict type-check               │
│  - ESLint + Prettier                          │
│  - 循环依赖检测 (madge)                        │
│  - 未使用依赖检测 (depcheck)                   │
└──────────────────┬───────────────────────────┘
                   │ 通过
                   ▼
┌──────────────────────────────────────────────┐
│ Stage 2: 单元测试 (5 min)                      │
│  - Jest / Vitest                              │
│  - 覆盖率 ≥ 90%                               │
│  - 覆盖率报告上传                              │
└──────────────────┬───────────────────────────┘
                   │ 通过
                   ▼
┌──────────────────────────────────────────────┐
│ Stage 3: 集成测试 (8 min)                      │
│  - Agent 协作流程测试                          │
│  - 工具链端到端测试                            │
│  - 安全规则集验证                              │
└──────────────────┬───────────────────────────┘
                   │ 通过
                   ▼
┌──────────────────────────────────────────────┐
│ Stage 4: 构建 (10 min)                         │
│  - iOS: xcodebuild + Fastlane                 │
│  - Android: gradle + Fastlane                 │
│  - 代码混淆 + 资源压缩                         │
│  - 生成符号表 (上传 Sentry)                    │
└──────────────────┬───────────────────────────┘
                   │ 通过
                   ▼
┌──────────────────────────────────────────────┐
│ Stage 5: 部署到预发布 (手动触发)               │
│  - iOS: TestFlight                            │
│  - Android: Internal Testing Track            │
│  - E2E 烟雾测试自动执行                        │
└──────────────────┬───────────────────────────┘
                   │ 通过
                   ▼
┌──────────────────────────────────────────────┐
│ Stage 6: 发布到生产 (审批后)                    │
│  - iOS: App Store Connect                     │
│  - Android: Google Play Console               │
│  - 分阶段推出 (20% → 50% → 100%)              │
└──────────────────────────────────────────────┘
```

### 1.2 分支策略

| 分支 | 用途 | 保护 |
|------|------|------|
| `main` | 生产代码 | 禁止直接 push，需 PR + 审批 + CI 通过 |
| `develop` | 开发主线 | 需 PR + CI 通过 |
| `feature/*` | 功能分支 | 从 develop 创建，合并到 develop |
| `hotfix/*` | 紧急修复 | 从 main 创建，合并到 main + develop |
| `release/*` | 发布准备 | 从 develop 创建，合并到 main |

### 1.3 构建产物

| 平台 | 格式 | 大小目标 |
|------|------|----------|
| iOS | .ipa (App Store) | < 80MB |
| Android | .aab (App Bundle) | < 50MB |
| 符号表 | .dSYM / mapping.txt | 上传 Sentry |

## 2. 环境配置

### 2.1 环境矩阵

| 配置项 | 开发 (Dev) | 预发布 (Staging) | 生产 (Production) |
|--------|-----------|-----------------|-------------------|
| **云端 AI API** | Mock 服务器 | 测试 API Key | 生产 API Key |
| **数据库** | 未加密 SQLite | 加密 SQLite (测试密钥) | 加密 SQLite (生产密钥) |
| **WebDAV** | 本地测试服务器 | 测试 WebDAV 服务器 | 用户自配 |
| **日志级别** | DEBUG | INFO | WARN |
| **崩溃上报** | 禁用 | Sentry (测试项目) | Sentry (生产项目) |
| **分析统计** | 禁用 | 禁用 | 可选 (需用户授权) |
| **热更新** | 启用 | 启用 | 启用 (仅 JS Bundle) |
| **代码混淆** | 禁用 | 启用 | 启用 |

### 2.2 配置管理

```typescript
// 配置通过 .env 文件 + 构建时注入
interface AppConfig {
  env: 'development' | 'staging' | 'production';
  
  // AI API
  aiApiKey: string;
  aiBaseUrl: string;
  aiDefaultModel: string;
  aiFallbackModel: string;
  
  // 数据库
  databaseName: string;
  databaseEncryptionEnabled: boolean;
  
  // 安全
  guardianEnabled: boolean;
  maxToolCallsPerRequest: number;
  dialogTimeoutMs: number;
  
  // 监控
  sentryDsn: string;
  analyticsEnabled: boolean;
  
  // 更新
  hotUpdateEnabled: boolean;
  hotUpdateBaseUrl: string;
}
```

## 3. 应用分发

### 3.1 App Store 上架要求

| 检查项 | 要求 |
|--------|------|
| 隐私清单 (PrivacyInfo.xcprivacy) | 列出所有使用的 API 及原因 |
| App Tracking Transparency | 不使用追踪，无需弹窗 |
| 加密声明 | 使用 AES-256，需提交 ERN 年度自我分类报告 |
| 最低版本 | iOS 15+ |
| 截屏 | 5.5" + 6.5" (或 6.7") 各 5 张 |

### 3.2 Google Play 上架要求

| 检查项 | 要求 |
|--------|------|
| 隐私政策链接 | 必须提供 |
| 数据安全声明 | 列出数据收集和使用 |
| 目标 API 级别 | ≥ 33 (Android 13) |
| 最低版本 | Android 12+ |
| App Bundle | .aab 格式 |

### 3.3 分阶段发布

| 阶段 | 比例 | 观察期 | 回滚条件 |
|------|------|--------|----------|
| Canary | 5% | 24h | 崩溃率 > 0.5% |
| Beta | 20% | 48h | 崩溃率 > 0.3% |
| Wide | 100% | — | 崩溃率 > 0.2% |

## 4. 数据库迁移

### 4.1 迁移策略

```
App 启动
  │
  ▼
检测当前数据库版本 (PRAGMA user_version)
  │
  ▼
有新迁移? ── 否 ──→ 正常启动
  │
  是
  ▼
┌────────────────────────────────┐
│ 迁移前自动备份                   │
│ (复制数据库文件到 backup/ 目录)  │
└────────────┬───────────────────┘
             │
             ▼
┌────────────────────────────────┐
│ 在事务中执行迁移脚本              │
│ BEGIN → 逐条执行 → 验证 → COMMIT│
│ 失败 → ROLLBACK → 恢复备份      │
└────────────┬───────────────────┘
             │
             ▼
更新 PRAGMA user_version
  │
  ▼
正常启动
```

### 4.2 迁移脚本规范

```sql
-- 文件名: migrations/20250115_120000_add_approval_log.sql
-- 描述: 新增审批日志表
-- 版本: 2 → 3

BEGIN;

CREATE TABLE IF NOT EXISTS approval_log (
  id TEXT PRIMARY KEY,
  tool_name TEXT NOT NULL,
  risk_score INTEGER NOT NULL,
  risk_level TEXT NOT NULL,
  user_decision TEXT NOT NULL,
  decision_time_ms INTEGER,
  created_at TEXT NOT NULL
);

CREATE INDEX IF NOT EXISTS idx_approval_log_created 
  ON approval_log(created_at);

COMMIT;
```

### 4.3 迁移版本管理

```sql
-- 每个迁移文件必须更新版本
PRAGMA user_version = 3;
```

| 版本 | 变更 | 迁移文件 |
|------|------|----------|
| 1 | 初始 schema | 001_init.sql |
| 2 | 新增 audit_log | 002_add_audit_log.sql |
| 3 | 新增 approval_log | 003_add_approval_log.sql |
| ... | ... | ... |

## 5. 监控与告警

### 5.1 关键指标

| 指标 | 采集方式 | 告警阈值 |
|------|----------|----------|
| 崩溃率 | Sentry | > 0.2% → 告警 |
| ANR 率 (Android) | Firebase Crashlytics | > 0.1% → 告警 |
| AI API 成功率 | 客户端埋点 | < 95% → 告警 |
| AI API P99 延迟 | 客户端埋点 | > 8s → 告警 |
| 工具调用错误率 | 本地 audit_log 统计 | > 5% → 应用内提示 |
| Token 消耗速率 | 本地计数器 | > 预算 80% → 用户警告 |
| 数据库大小 | 本地检测 | > 500MB → 提示归档 |
| 备份成功率 | 本地备份日志 | < 90% (3 次失败) → 提示用户 |

### 5.2 崩溃上报

```
崩溃发生
  │
  ▼
捕获 (Sentry SDK)
  ├─ 堆栈 + 设备信息 + 用户操作序列
  ├─ 去除了包含敏感数据的局部变量
  └─ 上传到 Sentry
  │
  ▼
Sentry 自动分组 + 分配优先级
  │
  ▼
Slack / 邮件通知 → 开发者处理
```

### 5.3 性能监控

```
App 启动
  │
  ▼
性能打点 (自定义埋点)
  ├─ app_cold_start_ms: 冷启动耗时
  ├─ db_query_ms: 数据库查询耗时 (P50/P95/P99)
  ├─ agent_response_ms: Agent 响应耗时
  ├─ chart_render_ms: 图表渲染耗时
  └─ backup_upload_ms: 备份上传耗时
  │
  ▼
本地聚合 (每日) → 可选上传分析服务
```

## 6. 热更新 (CodePush / 自建)

### 6.1 适用内容

| 可更新 | 不可更新 |
|--------|----------|
| JS Bundle (业务逻辑) | 原生代码 (.so / .framework) |
| Agent 提示词模板 | 数据库 schema 变更 |
| 分类规则模板 | 权限变更 |
| UI 样式调整 | 新的原生模块 |

### 6.2 更新流程

```
App 启动 / 前台切换
  │
  ▼
检查更新 (CodePush / 自建)
  │
  有更新?
  │
  ├─ 否 → 正常启动
  │
  └─ 是
      │
      ▼
  下载更新包
      │
      ▼
  校验签名 (SHA-256)
      │
      ▼
  安装 (静默)
      │
      ▼
  下次冷启动生效
```

## 7. 运维检查清单

### 7.1 发布前检查

- [ ] CI 全部通过 (静态检查 + 单元 + 集成)
- [ ] E2E 烟雾测试通过
- [ ] 崩溃率无回归
- [ ] 数据库迁移测试通过 (空库 → 最新 + 旧版升级)
- [ ] 安全扫描通过 (Guardian 规则集)
- [ ] 隐私清单更新
- [ ] 更新日志准备
- [ ] 回滚方案确认

### 7.2 发布后监控 (24h / 48h / 7d)

- [ ] 崩溃率在阈值内
- [ ] ANR 率在阈值内
- [ ] AI API 成功率正常
- [ ] Token 消耗无异常飙升
- [ ] App Store / Play Store 评分无异常下降
- [ ] 用户反馈无集中问题

### 7.3 定期维护

| 频率 | 任务 |
|------|------|
| 每日 | 检查崩溃率、API 成功率 |
| 每周 | 检查依赖更新 (npm/cocoapods/gradle) |
| 每月 | 审计日志归档、过期数据清理 |
| 每季度 | 安全渗透测试、性能基准测试 |
| 每半年 | 灾难恢复演练 |
