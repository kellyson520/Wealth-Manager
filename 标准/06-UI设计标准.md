# UI 设计标准 V1.0

> 来源：AI 原生对话式记账系统 最终汇总方案 V4.0 — 扩展
> 适用范围：纯对话界面设计、图表卡片规范、交互模式、视觉系统

---

## 1. 设计总则

### 1.1 核心设计理念

| 原则 | 说明 | 设计决策 |
|------|------|----------|
| **对话即界面** | 零菜单、零表单、零导航栏 | 所有功能入口为聊天输入框 |
| **AI 主动服务** | AI 不等用户问，主动推送洞察 | 启动时展示今日概览卡片 |
| **渐进式呈现** | 信息按重要性分层展示 | 摘要 → 详情 → 原始数据，逐层展开 |
| **即时反馈** | 每次操作 < 200ms 给反馈 | 输入即联想，发送即响应，骨架屏过渡 |
| **可信赖感** | 财务数据必须让人放心 | 关键操作有确认、金额有强调、异常有警示 |

### 1.2 反模式（禁止的设计）

- ❌ 传统侧边栏导航菜单
- ❌ 多级表单页面
- ❌ 模态弹窗阻断对话流
- ❌ 纯文本长列表（账单列表等）
- ❌ AI 回复无差异化样式（看不出是 AI 说的还是系统提示）

## 2. 对话界面规范

### 2.1 布局结构

```
┌─────────────────────────────────┐
│  Status Bar (时间/电量/网络)      │
├─────────────────────────────────┤
│  Title Bar (当前上下文标签)       │  ← 可折叠
│  "记账模式" | "分析模式" | ...   │
├─────────────────────────────────┤
│                                 │
│  消息列表 (Message List)         │
│  ┌──────────────────────────┐   │
│  │ 用户消息气泡 (右对齐)      │   │
│  └──────────────────────────┘   │
│  ┌──────────────────────────┐   │
│  │ AI 消息气泡 (左对齐)      │   │
│  │ 可嵌套：图表卡片/确认卡片  │   │
│  └──────────────────────────┘   │
│  ┌──────────────────────────┐   │
│  │ 系统消息 (居中，灰色)      │   │
│  └──────────────────────────┘   │
│                                 │
├─────────────────────────────────┤
│  Quick Bar (快捷操作建议)        │  ← 3-5 个上下文相关按钮
│  [💡 记一笔] [📊 本月分析] ...  │
├─────────────────────────────────┤
│  Input Bar (输入区域)            │
│  [🎤语音] [📎附件] [✏️____] [📤] │
└─────────────────────────────────┘
```

### 2.2 消息气泡类型

| 类型 | 视觉特征 | 使用场景 |
|------|----------|----------|
| **用户消息** | 右对齐，主题色背景，白色文字 | 用户任何输入 |
| **AI 回复** | 左对齐，白色/浅灰背景，深色文字 | AI 的文本回复 |
| **AI 卡片** | 左对齐，圆角卡片，可包含图表/表格/按钮 | 数据展示、确认操作 |
| **系统消息** | 居中，灰色小字，无气泡 | 状态变更、错误提示、时间戳 |
| **确认卡片** | 居中/左对齐，黄色/橙色边框，带操作按钮 | Guardian 拦截、2 级操作确认 |
| **错误卡片** | 左对齐，红色左边框 | 操作失败、异常提示 |

### 2.3 气泡内 Markdown 渲染

AI 回复支持 Markdown 子集：

| 元素 | 支持 | 样式 |
|------|------|------|
| 加粗/斜体 | ✅ | 财务数字必须加粗 |
| 有序/无序列表 | ✅ | 缩进 16px |
| 行内代码 | ✅ | 灰色背景等宽字体 |
| 表格 | ✅ | 自适应列宽，金额右对齐 |
| 链接 | ❌ 不渲染 | 防止钓鱼，显示为纯文本 |
| 图片 | ❌ 不渲染 | 通过附件卡片展示 |
| HTML | ❌ 全部过滤 | 安全策略 |

## 3. 图表卡片规范

### 3.1 卡片结构

```
┌─────────────────────────────────┐
│ 📊 本月支出分类           [展开] │  ← 卡片标题栏
├─────────────────────────────────┤
│                                 │
│  ┌─────────────────────────┐    │
│  │                         │    │
│  │   ECharts 图表           │    │  ← WebView 沙箱渲染
│  │   (饼图/柱图/折线/...)    │    │
│  │                         │    │
│  └─────────────────────────┘    │
│                                 │
│  💡 AI 洞察：餐饮支出环比 +12%   │  ← 洞察摘要
├─────────────────────────────────┤
│  [📋 查看详细]  [📤 分享]       │  ← 卡片操作
└─────────────────────────────────┘
```

### 3.2 图表类型与配置

| 图表类型 | ECharts 类型 | 使用场景 |
|----------|-------------|----------|
| 分类饼图 | `pie` (环形) | 月度支出/收入构成 |
| 趋势折线图 | `line` | 分类趋势、净资产变化 |
| 月度对比柱图 | `bar` | 月度收支对比 |
| 桑基图 | `sankey` | 资金流向分析 |
| 雷达图 | `radar` | 多维财务健康度 |
| 热力图 | `heatmap` | 日历热力（每日支出） |
| 仪表盘 | `gauge` | 预算使用率 |
| 堆叠柱图 | `bar` (stacked) | 分类月度堆叠 |

### 3.3 图表安全沙箱

WebView 渲染 ECharts 必须满足：

```
┌──────────────────────────────────┐
│ 安全沙箱约束                       │
├──────────────────────────────────┤
│ ✅ 仅渲染 ECharts 配置 JSON       │
│ ❌ 禁止执行任意 JavaScript         │
│ ❌ 禁止网络请求                    │
│ ❌ 禁止访问本地存储                │
│ ❌ 禁止跳转 URL                   │
│ ✅ 仅允许 echarts CDN 脚本加载     │
│    (打包到本地 assets，不联网)     │
└──────────────────────────────────┘
```

- 图表配置 JSON 在注入 WebView 前须经 `sanitize_chart_config` 校验
- 校验规则：拒绝含 `<script>`、`eval`、`Function`、`javascript:` 的配置
- JSON schema 白名单校验，只允许 echarts 已知配置项

## 4. 确认卡片设计

### 4.1 确认卡片结构（Guardian 插入）

```
┌═══════════════════════════════════┐
│ ⚠️ 安全提醒                       │
├───────────────────────────────────┤
│                                   │
│  检测到该笔支出 ¥5,000.00         │
│  超过历史月均值 3 倍              │
│                                   │
│  ┌─────────────────────────────┐  │
│  │ 商户：XX 数码城              │  │
│  │ 金额：¥5,000.00             │  │
│  │ 分类：电子产品               │  │
│  │ 日期：2025-01-15             │  │
│  └─────────────────────────────┘  │
│                                   │
│  [✅ 确认记账]  [✏️ 修改]  [❌ 取消] │
└═══════════════════════════════════┘
```

### 4.2 确认卡片类型

| 级别 | 图标 | 边框色 | 按钮 | 场景 |
|------|------|--------|------|------|
| 提醒 | ℹ️ | 蓝色 | [知道了] | 1级连续操作超过 5 次 |
| 确认 | ⚠️ | 橙色 | [确认] [取消] | 1级金额异常、批量操作 |
| 警告 | 🔴 | 红色 | [确认] [修改] [取消] | 2级操作、安全熔断 |
| 二次验证 | 🔐 | 红色 | [输入密码确认] [取消] | clear_all_data 等极端操作 |

### 4.3 确认卡片行为

- 卡片出现在对话流中，不阻塞 UI（非模态）
- 在用户做出选择前，对应操作处于"待确认"状态
- 超过 **60 秒**未响应，自动取消并记录审计日志
- 卡片可附带"信任此商家，后续不再提醒"选项

## 5. 快捷操作栏 (Quick Bar)

### 5.1 上下文感知建议

Quick Bar 根据当前对话状态动态变化：

| 对话状态 | 建议按钮 |
|----------|----------|
| 空闲（刚打开） | [💡 记一笔] [📊 今日概览] [🔍 查账] |
| 刚记完一笔 | [➕ 再记一笔] [🏷️ 加标签] [📊 本月统计] |
| 查看报表中 | [📤 导出] [📅 切换月份] [🎯 设预算] |
| 安全警告后 | [🔍 查看详情] [📋 查看规则] [⚙️ 调整阈值] |
| 导入文件后 | [✅ 全部确认] [🏷️ 批量分类] [📊 导入分析] |

### 5.2 交互规范

- 最多显示 **5 个**按钮，超出折叠为"更多"
- 点击后按钮变为加载态（spinner + 禁用）
- 建议按钮可被用户长按拖拽排序
- 支持通过设置固定常用按钮

## 6. 输入区域规范

### 6.1 输入框功能

```
┌──────────────────────────────────────────┐
│ [🎤] [📎] [✏️______________________________] [📤] │
│                                          │
│ 联想区域（按输入实时匹配）:                 │
│ ┌──────────────────────────────────────┐ │
│ │ 💡 "午饭 30" → 记账：午餐 ¥30         │ │
│ │ 🔍 "查一下" → 搜索最近账单            │ │
│ │ 📊 "分析" → 生成月度报表              │ │
│ └──────────────────────────────────────┘ │
└──────────────────────────────────────────┘
```

| 功能 | 触发方式 | 说明 |
|------|----------|------|
| 文本输入 | 点击输入框 | 支持多行，自动增高 |
| 语音输入 | 点击 🎤 | 本地语音识别，不传云端 |
| 附件上传 | 点击 📎 | 图片(OCR)、CSV/Excel、PDF |
| 发送 | 点击 📤 或回车 | 发送后输入框清空，焦点保持 |
| 联想提示 | 实时输入匹配 | 本地 NLU 意图预测，不消耗 Token |

### 6.2 输入联想引擎

- 纯本地实现，响应时间 < 50ms
- 基于 Trie 树 + 意图模板匹配
- 支持中英文混合输入
- 联想结果按频率 + 上下文排序

## 7. 加载与过渡状态

### 7.1 加载状态

| 场景 | 表现形式 | 超时处理 |
|------|----------|----------|
| AI 思考中 | 跳动省略号 "···" | 30s 超时提示 |
| 工具执行中 | 骨架卡片（灰色占位块） | 5s 超时提示 |
| 图表渲染中 | 骨架图（灰色圆形/柱形） | 3s 降级为表格 |
| 数据导入中 | 进度条 + 行数计数器 | 无超时，可后台执行 |
| 备份上传中 | 通知栏进度 | 完成后静默 |

### 7.2 空状态

| 场景 | 展示 |
|------|------|
| 无账单记录 | "📝 还没有记账记录，跟我说'午饭花了30块'开始吧" |
| 搜索无结果 | "🔍 没有找到相关账单，试试换个关键词" |
| 无预算设置 | "🎯 还没有设定预算，要不要现在设置一个？" |
| 无备份记录 | "☁️ 还没有备份过，配置 WebDAV 自动保护数据" |

### 7.3 错误状态

| 错误类型 | UI 表现 | 用户操作 |
|----------|---------|----------|
| 网络错误 | "🌐 网络连接失败" + 重试按钮 | [重试] [离线模式] |
| AI 超时 | "⏰ AI 响应超时" + 简化处理结果 | [重试] [本地处理] |
| 工具异常 | "⚠️ 操作失败：[原因]" + 错误代码 | [重试] [查看详情] |
| 数据异常 | "🔴 检测到数据异常" + Guardian 提醒 | [修复] [查看日志] |

## 14. 组件级规范

### 14.1 按钮

| 变体 | 样式 | 尺寸 | 使用场景 |
|------|------|------|----------|
| **主按钮** | 主题色填充 + 白色文字 | H: 44pt, 圆角 12px | 卡片中的主要操作（确认、发送） |
| **次按钮** | 主题色边框 + 透明底 | H: 44pt, 圆角 12px | 卡片中的次要操作（取消、跳过） |
| **文字按钮** | 主题色文字，无背景 | H: 32pt | 行内操作（查看详情、修改） |
| **危险按钮** | 红色填充 | H: 44pt, 圆角 12px | 删除、清除、撤销 |
| **胶囊按钮** | 浅灰底 + 深色文字 | H: 32pt, 圆角 full | Quick Bar 快捷建议 |
| **图标按钮** | 仅图标，44×44pt 触控区 | 44×44pt | 输入框旁：语音、附件 |

**按钮状态：**

| 状态 | 视觉 |
|------|------|
| Default | 正常色 |
| Pressed | 亮度降低 10% 或 scale 0.97 |
| Loading | Spinner 替换文字，按钮等宽 |
| Disabled | 30% 透明度，不可点击 |
| Focused (键盘) | 2px 主题色外环 |

### 14.2 输入框

```
┌──────────────────────────────────────────────┐
│                                              │
```
