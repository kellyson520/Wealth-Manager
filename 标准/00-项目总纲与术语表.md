# 项目总纲与术语表 V1.0

> 项目名称：Wealth Manager（AI 原生对话式记账系统）
> 代号：WM
> 版本：V4.0

---

## 1. 项目愿景

### 1.1 一句话定义

一款**完全由对话驱动**的个人财务智能体——用户用自然语言完成所有财务操作，App 没有传统界面。

### 1.2 我们要解决的问题

| 痛点 | 现有方案 | 我们的方案 |
|------|----------|------------|
| 记账太麻烦，坚持不下去 | 手动填表、选分类 | 说话即记账，AI 自动分类 |
| 数据看不懂 | 饼图柱图千篇一律 | AI 解读趋势 + 主动建议 |
| 担心隐私泄露 | 数据全上传云端 | 本地优先，云端脱敏后仅辅助推理 |
| AI 会乱操作 | 传统 AI 无安全边界 | Guardian Agent + 权限分级 + 熔断 |
| 换手机数据丢失 | 依赖厂商云同步 | WebDAV 自主备份 + 端到端加密 |

### 1.3 目标用户画像

| 画像 | 特征 | 核心需求 |
|------|------|----------|
| **记账新手** | 想记账但坚持不了 3 天 | 极低门槛，说话即记 |
| **月光族** | 不知道钱花哪了 | 自动分析 + 预算警告 |
| **理财进阶者** | 多账户、多币种 | 资产全视图 + 深度分析 |
| **隐私敏感者** | 不信任云端记账 APP | 本地存储 + WebDAV 自建备份 |
| **效率控** | 习惯语音/快捷指令 | Siri 集成 + 快捷指令 |

## 4. 术语表

### 4.1 系统核心概念

| 术语 | 英文 | 定义 |
|------|------|------|
| **智能体** | Agent | 独立的 AI 功能模块，有明确的职责边界和运行位置 |
| **Master Agent** | — | 唯一入口智能体，负责意图解析、任务分发、结果汇总 |
| **Ledger Agent** | — | 负责记账、查账、文件导入、OCR 等账单操作 |
| **Analyst Agent** | — | 负责统计分析、趋势检测、图表配置生成 |
| **Coach Agent** | — | 负责预算建议、储蓄挑战、情绪陪伴、成就激励 |
| **Guardian Agent** | — | 纯本地安全智能体，负责安全扫描、行为拦截、数据脱敏 |
| **工具** | Tool | 本地可调用的功能单元，Agent 通过调用工具完成操作 |
| **工具注册表** | Tool Registry | 所有工具的注册和发现中心 |
| **技能** | Skill | 可插拔的功能扩展包，用户可选安装 |
| **记忆** | Memory | 系统对用户偏好和行为的持久化记录 |

### 4.2 安全相关

| 术语 | 定义 |
|------|------|
| **权限级别** | 0级(Safe)/1级(Write)/2级(Sensitive)，控制工具调用是否需要确认 |
| **确认卡片** | Guardian 插入对话流的交互卡片，要求用户确认操作 |
| **安全模式** | 熔断后的状态，所有写操作需逐一确认 |
| **熔断** | 紧急中止当前操作并进入安全模式的机制 |
| **哈希链** | 每条账单包含前一条的哈希，形成不可篡改的链式结构 |
| **脱敏** | 将敏感数据转换为不可识别形式的过程 |
| **Output Guard** | AI 回复展示前的最终安全过滤器 |

### 4.3 数据相关

| 术语 | 定义 |
|------|------|
| **账单** | Bill，单条收支记录 |
| **分类规则** | Classification Rule，商户名 → 分类的映射规则 |
| **资产账户** | Asset Account，用户的各种资金账户 |
| **哈希指纹** | 每条账单的唯一哈希标识 |
| **向量嵌入** | Embedding，记忆文本的向量表示，用于相似度检索 |
| **用户画像** | User Profile，用户偏好和行为的参数化描述 |

### 4.4 交互相关

| 术语 | 定义 |
|------|------|
| **对话流** | Conversation Flow，聊天界面的消息序列 |
| **气泡** | Bubble，对话流中的单个消息容器 |
| **卡片** | Card，可嵌套图表/表格/按钮的富交互气泡 |
| **Quick Bar** | 输入框上方的上下文感知快捷按钮栏 |
| **意图路由** | Intent Routing，Master 根据用户输入分发到目标 Agent |
| **上下文** | Context，多轮对话中的历史和状态信息 |

### 4.5 部署相关

| 术语 | 定义 |
|------|------|
| **WebDAV** | 基于 HTTP 的文件管理协议，用于用户自主备份 |
| **端到端加密** | E2EE，数据在客户端加密后上传，服务器无法解密 |
| **SQLCipher** | SQLite 的加密扩展 |
| **Keychain / Keystore** | iOS / Android 的系统级安全密钥存储 |

## 6. 关键设计决策记录

| 决策 | 理由 | 日期 |
|------|------|------|
| 纯对话界面，不设传统菜单 | 降低记账门槛，差异化竞争 | 设计初期 |
| 五大 Agent 而非单体 AI | 职责分离，安全隔离，可独立升级 | 设计初期 |
| 本地工具为主，云端辅助推理 | 隐私优先，精确性保证 | 设计初期 |
| WebDAV 而非自建云存储 | 用户数据自主可控，降低运营成本 | 设计初期 |
| 链式哈希而非区块链 | 轻量防篡改，不需要分布式共识 | 设计初期 |
| ECharts WebView 沙箱渲染 | AI 动态生成图表，安全可控 | 设计初期 |
| 无云推送，纯本地通知 | 隐私优先，无服务器依赖 | 设计初期 |
