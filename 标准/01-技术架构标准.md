# 技术架构标准 V1.0

> 来源：AI 原生对话式记账系统 最终汇总方案 V4.0
> 适用范围：系统架构设计、技术选型、组件通信

---

## 1. 架构分层标准

系统采用四层架构，层间通过标准化接口通信，严禁跨层调用。

```
┌────────────────────────────────────────────────────┐
│                   表现层 (UI Layer)                  │
│  纯对话界面 (Flutter/React Native)                   │
│  + 系统 WebView 图表沙箱 (ECharts)                   │
│  + 桌面/锁屏小组件 + Siri/Google 快捷指令             │
└───────────────────────┬────────────────────────────┘
                        │
┌───────────────────────▼────────────────────────────┐
│              智能体协作层 (Agent Orchestration)       │
│  Master Agent (意图路由)                             │
│  Ledger Agent | Analyst Agent | Coach Agent | Guardian Agent │
└───────────────────────┬────────────────────────────┘
                        │
┌───────────────────────▼────────────────────────────┐
│              本地工具调用层 (Tool Executor)            │
│  80+ 工具：账单/资产/债务/报销/预算/图表/安全/备份...  │
└───────────────────────┬────────────────────────────┘
                        │
┌───────────────────────▼────────────────────────────┐
│              本地数据与引擎层 (Local Core)             │
│  加密 SQLite | 向量库 | 规则引擎 | 端侧模型(可选)     │
│  哈希链 | 记忆系统 | 人格参数 | Skill 注册表         │
└───────────────────────┬────────────────────────────┘
                        │ (仅必要且脱敏后)
                        ▼
┌────────────────────────────────────────────────────┐
│                云端 AI API (GPT-4o / Claude 等)      │
└────────────────────────────────────────────────────┘
```

### 1.1 各层职责

| 层级 | 职责 | 约束 |
|------|------|------|
| 表现层 | 纯对话 UI 渲染、WebView 图表展示、小组件 | 不持有业务逻辑，仅做展示和事件传递 |
| 智能体协作层 | 意图解析、任务分发、结果汇总 | Master 为唯一入口，Agent 间通过消息总线通信 |
| 本地工具调用层 | 工具注册、执行、结果返回 | 所有工具必须幂等可重试，支持超时终止 |
| 本地数据与引擎层 | 持久化存储、向量检索、规则匹配、加密 | SQLite 必须加密，哈希链不可变 |

### 1.2 云端通信原则

- **仅必要且脱敏后**才向云端 API 发送数据
- 发送前必须经 Guardian Agent 脱敏处理
- 云端仅承担推理任务，不持久化任何用户数据

## 2. 技术选型标准

| 组件 | 选型要求 | 说明 |
|------|----------|------|
| 客户端框架 | Flutter 或 React Native | 跨平台，支持 WebView 嵌入 |
| 本地数据库 | SQLite (加密) | 轻量、可靠，支持全文搜索 |
| 向量存储 | 本地向量库 | 用于记忆相似度检索 |
| 图表渲染 | ECharts (WebView 沙箱) | AI 输出配置 JSON，WebView 渲染 |
| 加密算法 | AES-256 | 端到端加密备份 |
| 哈希算法 | SHA-256 链式哈希 | 账单防篡改 |
| 本地通知 | 系统本地通知 API | 无云推送依赖 |
| 云端 AI | GPT-4o / Claude API | 仅用于复杂推理 |

---

## 3. 组件通信标准

### 3.1 Agent 间通信

- 所有 Agent 通过 **消息总线 (Message Bus)** 通信
- Master Agent 为唯一入口，接收用户输入后路由至目标 Agent
- Agent 间传递的数据必须为结构化 JSON，不得传递原始用户文本

### 3.2 UI 与 Agent 通信

- UI 层仅通过 Master Agent 接口与后端交互
- 图表渲染：Agent 输出 ECharts 配置 JSON → UI 层 WebView 渲染
- 确认卡片：Guardian Agent 可主动插入确认卡片到对话流中

### 3.3 工具调用规范

- 工具注册在 Tool Registry 中，按命名空间分组
- 每次调用记录写入 audit_log
- 工具执行超时：单次 ≤ 5s，超时自动终止并返回错误

### 4.2 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| Agent 文件 | `{name}.agent.ts` | `master.agent.ts` |
| 工具文件 | `{verb}_{noun}.tool.ts` | `add_bill.tool.ts` |
| 数据库迁移 | `{timestamp}_{description}.sql` | `20250101_init.sql` |
| 类型定义 | PascalCase | `BillRecord`, `AgentMessage` |
| 工具名称 | snake_case | `add_bill`, `search_bills` |

## 5. 部署与运行环境

- **客户端**：iOS 15+ / Android 12+
- **本地存储**：≥ 100MB 可用空间
- **网络**：WiFi 或蜂窝网络（云端 AI 调用需网络）
- **权限**：通知权限（定时提醒）、后台任务（自动化）

## 6. 缓存策略

### 6.1 多级缓存

```
┌─────────────────────────────────────┐
│  L1: 内存缓存 (LRU, 最大 32MB)       │
│  热点数据：最近 50 条账单、常用规则   │
│  命中率目标：≥ 80%                   │
│  TTL: 会话级                         │
└──────────────┬──────────────────────┘
               │ 未命中
               ▼
┌─────────────────────────────────────┐
│  L2: SQLite 数据库                    │
│  全量数据 + FTS5 索引                │
│  查询目标：< 50ms (索引) / < 200ms   │
└──────────────┬──────────────────────┘
               │ 未命中 (极少)
               ▼
┌─────────────────────────────────────┐
│  L3: WebDAV 远程备份 (只读恢复)       │
│  仅恢复场景访问                      │
└─────────────────────────────────────┘
```

### 6.2 缓存键策略

| 数据类型 | 缓存键 | 失效策略 |
|----------|--------|----------|
| 最近账单 | `bills:recent:50` | 新记账时 prepend，超出 50 条 pop |
| 分类规则 | `rules:all` | 规则变更时全量刷新 |
| 用户画像 | `profile:singleton` | 变更时更新 |
| 汇率 | `fx:{from}:{to}` | TTL 24h |
| 图表配置 | `chart:{type}:{month}` | 月度数据变更时失效 |

### 6.3 内存预算

| 组件 | 内存预算 | 备注 |
|------|----------|------|
| L1 缓存 | 32MB | 可在设置中调整 |
| 向量嵌入 | 16MB | 记忆系统 |
| Agent 运行时 | 32MB | 消息 + 上下文 |
| UI 渲染 | 40MB | Flutter/RN 引擎 |
| 其他 | 30MB | 系统开销 |
| **总计** | **≤ 150MB** | 运行时峰值 |

## 7. 离线能力

### 7.1 离线功能矩阵

| 功能 | 离线可用 | 说明 |
|------|:---:|------|
| 记账 (文本) | ✅ | 纯本地 NLU + DB |
| 记账 (语音) | ✅ | 本地语音识别 |
| 查询账单 | ✅ | 本地 SQLite |
| 统计分析 | ✅ | 本地聚合 |
| 规则匹配 | ✅ | 本地规则引擎 |
| 图表渲染 | ✅ | 本地 ECharts (缓存数据) |
| AI 深度分析 | ❌ | 需云端 LLM |
| AI 规则学习 | ❌ | 需云端 LLM |
| WebDAV 备份 | ❌ | 需网络 |
| 汇率更新 | ❌ | 使用缓存汇率 |

### 7.2 离线到在线过渡

```
检测到网络恢复
  │
  ▼
1. 检查是否有待同步操作 (无 — 本系统无需同步)
2. 检查汇率缓存是否过期 → 是 → 后台静默更新
3. 恢复云端 AI 能力 → 用户无感知
4. 如有定时备份到期 → 自动触发 WebDAV 备份
```
