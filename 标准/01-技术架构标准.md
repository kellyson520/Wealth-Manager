## 1. 架构分层标准

系统采用 **六层 DDD 架构**，层间通过接口通信，严禁跨层调用聚合内部状态。

```
┌─────────────────────────────────────────────────────────┐
│              表现层 (Presentation / UI)                  │
│  ChatScreen / Cards (Bill/Chart/Confirm/Error/Tip)      │
│  WebView ECharts 沙箱 (src/ui/charts/)                   │
│  LogScreen                                              │
├─────────────────────────────────────────────────────────┤
│            应用服务层 (Application Services)              │
│  BillingService / AnalyticsService / CoachService        │
│  GuardianService / MasterOrchestrator                    │
│  (替代旧 Agent 直接工具调用，改为注入 Repository+EventBus) │
├─────────────────────────────────────────────────────────┤
│              领域层 (Domain Layer)                        │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐   │
│  │ billing  │ │ budget   │ │ asset    │ │gamification│  │
│  │ 聚合根:   │ │ 聚合根:   │ │ 聚合根:   │ │ 聚合根:    │  │
│  │ Bill     │ │BudgetPlan│ │Asset,Debt│ │Achievement │  │
│  │          │ │SavingsGoal│ │          │ │Streak      │  │
│  ├──────────┤ ├──────────┤ ├──────────┤ ├──────────┤   │
│  │analytics │ │automation│ │ rules    │ │ shared    │   │
│  │(只读上下文)│ │ 聚合根:   │ │ 聚合根:   │ │ Shared     │  │
│  │          │ │RecurrTask │ │ClassRule │ │ Kernel     │  │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘   │
├─────────────────────────────────────────────────────────┤
│            基础设施层 (Infrastructure)                    │
│  DomainEventBus / SQLite Repositories / Cache /          │
│  MessageBus / CircuitBreaker / HashChain / VectorStore   │
├─────────────────────────────────────────────────────────┤
│           Agent 适配层 (保留为兼容层)                      │
│  Master / Ledger / Analyst / Coach / Guardian            │
│  逐步迁移到 Application Service 注入模式                  │
├─────────────────────────────────────────────────────────┤
│             外部 API 层 (External)                        │
│  GPT-4o / Claude (仅脱敏后调用)                          │
└─────────────────────────────────────────────────────────┘
```

### 1.1 各层职责

| 层级 | 目录 | 职责 | 约束 |
|------|------|------|------|
| 表现层 | `src/ui/` | 纯对话 UI 渲染、WebView 图表、卡片组件 | 不持有业务逻辑，仅展示 DTO |
| 应用服务层 | `src/application/` | 用例编排：校验→调领域→持久化→发事件 | 不含 if/else 业务判断 |
| 领域层 | `src/domain/` | 聚合根、值对象、领域事件、Repository 接口 | 不依赖基础设施 |
| 基础设施层 | `src/infrastructure/` | Repository 实现、EventBus 实现、DB/Cache | 实现 Domain 层定义的接口 |
| Agent 适配层 | `src/agents/` | NLU 分类、意图路由、旧工具调用 | 逐步迁移到 Application Service |
| 外部 API | `src/core/cloud/` | LLM API 调用、函数调用转换 | 仅脱敏后发送 |

## 2. 领域驱动设计 (DDD) 规范

### 2.1 Bounded Context 矩阵

| Context | 目录 | 聚合根 | 所属表 | 独立性 |
|---------|------|--------|--------|:---:|
| **Billing** | `domain/billing/` | Bill | `bills`, `categories`, `tags`, `bill_tags` | ⭐⭐⭐ |
| **Budget** | `domain/budget/` | BudgetPlan, SavingsGoal | `budgets`, `savings_goals`, `user_profile` | ⭐⭐⭐ |
| **Asset** | `domain/asset/` | Asset, Debt | `assets`, `debts`, `repayments` | ⭐⭐⭐ |
| **Gamification** | `domain/gamification/` | Achievement, Streak | `achievements` | ⭐⭐⭐ |
| **Analytics** | `domain/analytics/` | (Read-only) | 无自有表 | ⭐⭐ |
| **Automation** | `domain/automation/` | RecurringTask | `recurring_tasks`, `shortcuts` | ⭐⭐⭐ |
| **Rules** | `domain/rules/` | ClassificationRule | `classification_rules` | ⭐⭐⭐ |
| **Shared** | `domain/shared/` | — | `audit_log`, `sync_state` | — |

### 2.2 聚合设计原则

```
聚合根 (Aggregate Root)
├── 持有 Value Objects (Money, DateRange, BillType)
├── 产出 Domain Events (BillRecorded, BudgetOverrun...)
├── 封装业务规则 (金额校验、超支检测、进度计算)
└── 禁止：直接引用另一个 Aggregate Root

Repository (接口在 domain/，实现在 infrastructure/)
├── save(aggregate) / findById(id) / delete(id)
├── 查询方法按需定义（search, aggregate, getCategoryTotals...）
└── 禁止：返回原始 SQL 行、拼接跨聚合 JOIN

Application Service (src/application/)
├── 构造注入 Repository + EventBus
├── 方法 = 一个完整用例 (recordBill, modifyBill, deleteBill)
└── 禁止：写 if/else 业务判断 → 那属于 Domain 层
```

### 2.3 领域事件目录

| Event | 发布者 | 订阅者 |
|-------|--------|--------|
| `BillRecorded` | Billing::Bill | Gamification(进度), Budget(检查超支), Automation(场景评估) |
| `BillModified` | Billing::Bill | Security(审计日志) |
| `BillDeleted` | Billing::Bill | Analytics(刷新缓存), Gamification(回退进度) |
| `BillCategoryCorrected` | Billing::Bill | Rules(记录纠正) |
| `BudgetLimitSet` | Budget::BudgetPlan | Analytics(更新图表) |
| `BudgetOverrun` | Budget::BudgetPlan | Automation(通知), UI(警告卡) |
| `SavingsGoalCompleted` | Budget::SavingsGoal | Gamification(解锁成就), Automation(通知) |
| `AchievementUnlocked` | Gamification::Achievement | Automation(通知), UI(动画) |
| `AssetValueChanged` | Asset::Asset | Analytics(净资产更新) |
| `DebtRepaid` | Asset::Debt | Analytics(负债更新) |
| `TaskTriggered` | Automation::RecurringTask | Infrastructure(执行任务) |
| `RuleMatched` | Rules::ClassificationRule | Billing(自动分类) |

**投递语义**: At-Least-Once，基于 `MessageBus` + 死信队列

## 4. 组件通信标准

### 4.1 消息总线 (MessageBus) — Agent 间通信

```typescript
// src/core/message-bus/
interface BusMessage {
  id: string;
  from: AgentId;                    // 'master' | 'ledger' | 'analyst' | 'coach' | 'guardian'
  to: AgentId | 'broadcast';
  type: 'request' | 'response' | 'event' | 'error';
  payload: Record<string, unknown>;
  correlationId?: string;           // 请求-响应关联
  createdAt: string;
}
```

**特性**: Pub/Sub 模式，广播/单播双模，死信队列 (SQLite 持久化)，重试 3 次

### 4.2 领域事件总线 (DomainEventBus) — Context 间解耦

```typescript
// src/infrastructure/events/
interface DomainEventBus {
  publish(event: DomainEvent): Promise<void>;
  publishAll(events: DomainEvent[]): Promise<void>;
  subscribe<T>(eventClass, handler): () => void;
}
```

**基于 MessageBus 实现**，eventType 作为路由键自动分发到订阅者

### 4.3 UI 与后端通信

```
ChatScreen
  → handleSend(text)
    → MasterOrchestrator (NLU 分类)
      → Application Service (用例编排)
        → Aggregate (业务规则)
          → DomainEvent (产出事件)
        → Repository (持久化)
        → EventBus.publishAll (广播事件)
      → 返回 DTO → ChatMessage.data → CardRenderer → 卡片渲染
```

## 5. 缓存架构

### 5.1 MemoryCache (src/core/cache/)

| 属性 | 值 |
|------|-----|
| 最大容量 | 500 条 |
| 默认 TTL | 30 分钟 |
| 淘汰策略 | LRU (最旧优先) |
| 熔断器 | 5 次连续穿透 → 60s 熔断 |

### 5.2 ToolCacheWrapper

仅对幂等工具 (`idempotent: true`) 自动缓存结果。缓存键 = `tool:{name}:{sorted_args}`

## 9. 反模式禁止清单 (DDD)

| # | 禁止 |
|---|------|
| 1 | 为单个字段创建 Value Object 类 |
| 2 | Aggregate 直接持有另一个 Aggregate 的引用 |
| 3 | Application Service 里写 if/else 业务判断 |
| 4 | Repository 返回 Domain Event |
| 5 | 跨 Context 的同步 RPC 调用 |
| 6 | SQLite Repository 里 JOIN 跨聚合查询 |
| 7 | 超过 3 层的类继承体系 |
| 8 | Repository 使用 `any` 泛型 |
